# **Тест-кейсы для микросервиса**
API содержит 4 ручки, по которым тестирование было разбито на группы:

1. **GET /api/1/item/{id}**

|  №  |        Описание кейса                                   |   Входные данные          | Ожидаемый результат |
|:----|:--------------------------------------------------------|:--------------------------|:--------------------|  
|1    |Проверить схему ответа на запрос по валидному `id`         | валидный id товара        | JSON верного формата|
|2    |Проверить схему ответа на запрос по неверному `id`         | строка, несуществующий id | JSON верного формата|
|3    |Проверить код статуса ответа по валидному `id`             | валидный id товара        |  status `200`         |
|4    |Проверить код статуса ответа по неверному `id`             | валидный id товара        |  status `400`         |
|5    |Проверить обработку вызова запроса с <br> неправильным методом| POST метод           |  status `405`         |

2. **GET /api/1/{sellerID}/item**

| №  |        Описание кейса                                   |   Входные данные                        | Ожидаемый результат | 
|:----|:-------------------------------------------------------|:----------------------------------------|:---------------------|  
|1   |Проверить схему ответа на запрос по валидному `id`         | валидный id товара                      | JSON верного формата |
|2   |Проверить схему ответа на запрос по неверному `id`        |строка,                              null| JSON верного формата |
|3   |Проверить код статуса ответа по валидному `id`             | валидный id товара                      |  status `200`          |
|3   |Проверить код статуса ответа по <br>валидному `id` продавца без товаров| валидный id товара                      |  status `404`          |
|4   |Проверить код статуса ответа по неверному `id`             |строка,                              null|   status `400`, `404`    |
|5   |Проверить обработку вызова запроса с <br> неправильным методом| POST метод                         |  status `405`          |

3. **GET /api/1/statistic/{id}**

| №  |        Описание кейса                                   |   Входные данные          | Ожидаемый результат |
|:----|:-------------------------------------------------------|:--------------------------|:--------------------|  
|1   |Проверить схему ответа на запрос по валидному `id`         | валидный id товара        | JSON верного формата|
|2   |Проверить схему ответа на запрос по неверному `id`         |строка, несуществующий id, null|JSON верного формата|
|3   |Проверить код статуса ответа по валидному `id`             | валидный id товара        |  status `200`         |
|4   |Проверить код статуса ответа по неверному `id`             |строка,                null|  status `400`         |
|5   |Проверить обработку вызова запроса с <br> неправильным методом| POST метод           |  status `405`         |
|6   |Проверить содержимое ответа на запрос  GET /item/{id}    | id созданного товара      |статистики совпадают |

4. POST /api/1/item

| №  |        Описание кейса                                   |   Входные данные          | Ожидаемый результат |
|:----|:-------------------------------------------------------|:--------------------------|:--------------------|  
|1   |Проверить схему ответа после создания объявления         |    валидный payload       | JSON верного формата|
|2   |Проверить схему ответа после создания объявления<br> с невалидными данными    | невалидный payload          | JSON верного формата|
|4   |Проверить обработку неверных payload                     | невалидные payload'ы (неверный <br>формат имени, id продавца, статистики)        |  status `400`         |
|5   |Проверить статус ответа после создания объявления        | валидный payload          |  status `200`         |
|6   |Проверить соответствие payload и ответа<br> на запрос GET /item/{id}| валидный payload        |  респонсы совпадают       |
