В инструкции ниже вы сможете найти структуру репозитория и инструкцию по установке и запуску решения.

## **Задание 1.**

В ходе выполнения первого задания я изучил скриншот сервиса AvitoTeam на наличие багов. В файле ```TASK1.md``` я составил баг-репорт, содержащий в себе скриншоты ошибок, их описание, приоритет и серьезность, а также обоснование присвоения того или иного уровня. 

## **Задание 2.**

В ходе выполнения второго задания я тестирование микросервиса, API которого содержит 4 запроса:
- Создать объявления (`POST /api/1/item`)
- Получить объявления по его идентификатору (`GET /api/1/item/{id}`)
- Получить все объявления по идентификатору продавца (`GET /api/1/{sellerId}/item`)
- Получить статистику по айтем id (`GET /api/1/statistic/{id}`)

### **Структура проекта**
[api.py](./api.py) - содержит в себе функции, реализующие взаимодействие микросервиса с тестами. \
[schemas.py](./schemas.py) - описывает JSON схемы респонсов, используется для валидации. \
[conftest.py](./conftest.py) - конфигурационный файл для пакета `pytest`, в котором содержатся глобальные настройки для тестов и фикстуры, необходимые для их работы. \
[requirements.txt](./requirements.txt) - список зависимостей. Используется для настройки окружения. \
[test_post_item.py](./test_post_item.py), [test_get_item_by_id.py](./test_get_item_by_id.py), [test_get_item_by_seller_id.py](./test_get_item_by_seller_id.py), [test_get_stats_by_item_id.py](./test_get_stats_by_item_id.py)  - файлы, реализующие логику тестов. \
[TESTCASES.md](./TESTCASES.md) - содержит описание тест-кейсов (входные данные, что проверяем и что ожидаем)
[BUGS.md](./BUGS.md) - содержит баг-репорт с оценкой приоритета и серьезности багов.

### **Настройка окружения и запуск тестов**

1. **Склонируйте этот репозиторий.** Для этого откройте терминал в папке, в которой хотите сохранить репозиторий, и выполните команду: 
    ```
    git clone https://github.com/xauvlad/avito-qa-assignment.git
    ```

2. **Установите `Python`.** Для этого перейдите на официальный сайт и следуйте инструкциям по установке на вашу операционную систему. 
    Убедитесь в корректной установке выполните в терминале команду: 
    **Windows:** 
    ```
    python --version
    ``` 
    **macOS / Linux:** 
    ```
    python3 --version
    ```
    Если команда выводит версию Python, вы можете переходить к следущему шагу.

3. **Создайте виртуальное окружение.** Перейдите в папку проекта и в терминале введите команды: 

    **Windows:**
    ```
    python -m venv <имя_виртуального_окружения>
    <имя_виртуального_окружения>\Scripts\activate
    ```
    **macOS / Linux:**
    ```
    python3 -m venv <имя_виртуального_окружения>
    source <имя_виртуального_окружения>/bin/activate
    ```

    При успешной активации окружения в терминале появится префикс `(<имя_виртуального_окружения>)`. 

4. **Установите зависимости.** Введите команду: 
      ```
      pip install -r requirements.txt
      ```

5. **Запустите тесты.**
    Для запуска **всех тестов** из корня проекта выполните: 
    ```
    pytest
    ```
    Чтобы запустить тесты по одной из ручек, выполните: 
    ```
    pytest <имя_файла.py>
    ```
    , например: <br> 
    ```
    pytest test_get_item_by_id.py
    ```
    Чтобы запустить отдельный тест, напишите его имя, например:
    ```
    pytest test_get_item_by_id.py::test_get_item_by_id_valid_schema
    ```
